Reference
Dataset & Usage Rules
Guidelines for optimal prediction accuracy across different datasets and use cases.
Works With Any Dataset
1
Any multi-class text classification task. The model reads labels dynamically from label_config.json — swap the model folder to change domains entirely.
2
Required columns for batch upload: title (mandatory), body (optional but improves accuracy).
3
English language only. The underlying RoBERTa checkpoint was trained on English corpora.
4
Maximum 1,000 rows per batch. Split larger datasets into chunks or use the API directly.
Accuracy Guidelines
5
Provide both title and body. Title-only input can reduce accuracy by 8–12% for ambiguous categories.
6
Do not pre-process input. The tokeniser handles whitespace, punctuation, and casing internally.
7
Confidence below 55% warrants human review. The question label in particular overlaps with bug reports.
8
Use the downloadable Excel for audit trails. Per-label confidence scores are included for every row.
Data Flow
Input
title + body
Tokeniser
Sentence pair (max 384)
RoBERTa
12-layer, 768 hidden
Classifier
Dense + softmax
Output
Label + scores
